## C'est quoi ce projet ?

On nous a confié le code source d'un site web laissé par un stagiaire 
parti de la startup fictive DevStart Agency. Notre mission : lire ce 
code, comprendre ce qu'il fait, identifier ses problèmes, et tout 
documenter clairement.

On n'a pas réécrit le code. On l'a analysé.

---

## Structure du dépôt
/original       → Le code source tel qu'on l'a reçu, sans aucune modification
/annotated      → Le même code avec nos commentaires ajoutés section par section
README.md       → Ce fichier

---

## Notre démarche

**Jour 1-2 — Lecture**  
On a lu tout le code de A à Z sans rien toucher. Chaque membre du groupe 
a annoté sa partie, puis on a mis en commun.

**Jour 3-4 — Analyse critique**  
On a listé les problèmes de lisibilité, de logique et d'organisation. 
On a aussi noté ce qui fonctionnait bien pour ne pas être injustes.

**Jour 5 — Rapport et présentation**  
Rédaction du rapport final et préparation des slides pour la présentation 
devant l'académie.

---

## Ce qu'on a trouvé (résumé)

- Les noms de classes CSS sont illisibles (`.b1`, `.c`, `.dd`...)
- Les liens de navigation sont abrégés (`acc`, `nous`, `serv`, `cont`)
- Les styles des cartes services et des témoignages sont dupliqués inutilement
- Le formulaire de contact ne fonctionne pas (bouton sans script)
- Le champ email est de type `text` au lieu de `type="email"`
- Aucune media query — le site ne s'adapte pas au mobile
- Pas de balises HTML sémantiques (`<nav>`, `<section>`, `<footer>`...)
- Copyright périmé : 2024 au lieu de 2026
